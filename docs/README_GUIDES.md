# QuickLog-Solo ガイドブック (開発・設定リファレンス)

本ドキュメントは、QuickLog-Solo の補助ツールである **QL-Category Editor** および **QL-Animation Studio** の操作方法と詳細な仕様をまとめたガイドです。初めて利用するユーザーがスムーズに操作でき、開発者がオンラインガイドを構築する際のリファレンスとなることを目的としています。

---

## 1. はじめに

QuickLog-Solo およびその補助ツールは、すべて**ブラウザ上のみで完結し、外部サーバーとは一切通信を行わないスタンドアローン形式**で動作します。この「純粋なローカル動作」により、ユーザーの作業データが外部に漏れることはありません。

- **QL-Category Editor**: カテゴリの一括編集、高度なタグ管理、並べ替えを広い画面で行うためのツール。
- **QL-Animation Studio**: 背景アニメーションを自ら作成・調整・テストするための開発環境。

これらのツール間のデータ連携（クリップボード、ファイルへの書き出し/読み込み）もすべてユーザーの手元で完結しており、プライバシーと永続性が担保されています。

---

## 2. QL-Category Editor 操作ガイド

カテゴリ（作業項目）の名称、色、アニメーション、タグを一括で管理します。

### 2.1. カテゴリリストの操作
- **追加と削除**:
    - **カテゴリ追加**: ヘッダーの `+` ボタンをクリック。
    - **ページ区切り追加**: リスト下部の「改ページを追加 (Add Page Break)」ボタンをクリック。
    - **削除**: 各項目のゴミ箱アイコン、または `more_vert` メニューから実行。
- **並べ替え**: 左端の `drag_indicator` をドラッグ＆ドロップ。移動先が青いライン（インジケーター）で表示されます。
- **複製**: `more_vert` メニューの「複製」を選択。名称に `(1)` などの連番が自動付与されます。

### 2.2. 選択と編集の挙動
選択状態によって、右側の詳細編集パネルで操作できる項目が異なります。

| 項目 | 単一選択時 | 複数選択時 |
| :--- | :--- | :--- |
| **名称** | 編集可能 | **編集不可** (空欄表示) |
| **タグ** | そのカテゴリのタグを表示・編集 | 選択された全カテゴリの**共通タグ（積集合）**を表示。追加・削除は全選択項目へ反映。 |
| **色** | そのカテゴリの色を表示・編集 | **選択範囲の先頭項目**の色を表示。変更は全選択項目へ反映。 |
| **アニメーション** | そのカテゴリの設定を表示・編集 | **選択範囲の先頭項目**の設定を表示。変更は全選択項目へ反映。 |
| **個別アクション** | 複製・削除ボタンが表示される | ボタンが非表示になる（ヘッダーの一括削除を使用） |

- **選択方法**:
    - **単一選択**: 項目をクリック。
    - **複数選択 (個別)**: `Ctrl` (Macは `Cmd`) を押しながらクリック。
    - **範囲選択**: 1つ選択後、`Shift` を押しながら別の項目をクリック。

### 2.3. 高度なタグ管理
- **タグの入力**: 文字入力後に `Enter` または `,`（カンマ）で確定。
- **Tag Box (タグボックス)**: 全カテゴリで使用中のユニークなタグを一覧表示。
    - **ドラッグ＆ドロップ**: タグをボックスからタグ入力欄へドラッグして追加できます。ドラッグ中は入力欄が点線で強調されます。
- **改ページ (Page Break)**: 名称や色、アニメーション、タグを持ちません。リスト上での並べ替え専用の項目です。

### 2.4. Undo/Redo (履歴管理)
最大50手まで遡ることが可能です。
- **テキスト編集のグループ化**: 名前の変更などは、入力開始から完了（フォーカスアウト）までを1つのアクションとして記録します。
- **ショートカット**:
    - `Ctrl + Z` (Mac: `Cmd + Z`): 元に戻す。
    - `Ctrl + Y` または `Ctrl + Shift + Z` (Mac: `Cmd + Shift + Z`): やり直し。

## 3. QL-Animation Studio 操作ガイド

背景アニメーションをゼロから構築、または既存サンプルを調整するための環境です。

### 3.1. 構成要素
- **Metadata**: 名前、作者、説明。多言語対応しており、言語を切り替えてそれぞれの翻訳を登録可能。
- **Configuration**:
    - **Mode**: 単一の描画エンジン上で動作する3つの描画モード（Canvas 2D / Sprite / Matrix）から選択。
    - **Exclusion Strategy**: UI要素を避けるアルゴリズム（`mask`: 隠す, `jump`: 飛び越える, `freedom`: 無視）。
    - **Rewindable**: タイムトラベル（巻き戻し）対応の有無。
- **Code Editor (コードエディタ)**: JavaScript ロジックを記述する中心領域。
    - **Variables**: クラス変数・定数。
    - **setup()**: 初期化。
    - **draw()**: フレーム毎の描画。
    - **Interaction**: クリック・マウス移動。

### 3.2. スマートエディタ機能
- **オートインデント**: `Enter` キーで改行した際、前の行のインデントを自動で引き継ぎます。
- **オートクローズ**: `{`, `[`, `(`, `"`, `'` を入力すると、対になる閉じ記号を自動挿入します。
- **検索・置換**: `Ctrl + F` (Mac: `Cmd + F`) で検索バーを表示し、現在のタブ内の文字列を検索・一括置換。
- **折り返し切り替え**: コードの折り返し（Word Wrap）を ON/OFF 可能。

### 3.3. テスト走行 (テープデッキ操作)
カセットデッキを模したインターフェースで動作を確認します。
- **再生 (▶)**: コードをコンパイルし実行。
- **速度調整**: 0.5x 〜 2.0x のレンジで調整可能。
- **エジェクト (⏏)**: スタジオの状態を初期化（サンプル選択解除）。

### 3.4. シミュレーターと計測
- **Exclusion Area**: プレビュー上の四角形をドラッグ・リサイズして UI 回避をテスト。
- **Metrics (計測値)**: Latency（遅延）、Density（密度）、Change Rate（変化率）をリアルタイム表示。
- **Console**: `console.log()` 出力やエラーを表示。

## 4. 本体アプリ (QuickLog-Solo) とのデータ連携

連携作業はすべて**ブラウザのクリップボードまたはローカルファイル保存**を通じて行われ、外部サーバーを経由することはありません。

### 4.1. カテゴリデータの連携 (Editor ↔ 本体)
- **Editor → 本体 (反映)**:
    1. Editor 右下の「コピー (Export)」をクリック（NDJSON形式がクリップボードへ）。
    2. 本体設定 ＞ 「カテゴリ」タブ ＞ 「貼り付け (Import)」をクリック。
    3. 追記または上書きを選択して実行（500msの遅延後、成功通知が表示）。
- **本体 → Editor (編集開始)**:
    1. 本体設定 ＞ 「カテゴリ」タブ ＞ 「コピー (Export)」をクリック。
    2. Editor 右下の「貼り付け (Import)」をクリック。

### 4.2. アニメーションデータの連携 (Studio ↔ 本体)
- **Studio → 本体 (利用)**:
    1. 「Download」ボタンで `.js` ファイルとしてローカルに保存。
    2. 公式リポジトリへ配置（詳細は「Prepare PR」参照）。
- **既存アニメーションの読み込み**:
    1. 「Upload」ボタンからローカルの `.js` または `.json` ファイルを選択して読み込み。
    2. サンプル選択メニューから既存のプリセットを読み込んで調整。

### 4.3. スキーマバリデーション
データ連携時は厳格なチェックが行われ、`kind` や `version` が不正なデータは拒否されます。一部に不備がある場合は、正常な分のみを取り込むかどうかの選択肢が表示されます。
