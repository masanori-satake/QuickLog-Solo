# QuickLog-Solo: テスト計画・ケース定義書

本プロジェクトにおけるテスト方針、自動テストの構成、および主要なテストケースについてまとめます。

## 1. テスト方針

- **自動化:** 全てのコアロジックと重要な UI 操作を自動テストでカバーします。
- **マルチレイヤー:** ユニットテスト (Jest), E2Eテスト (Playwright), およびアニメーション品質評価を組み合わせます。
- **多言語検証:** 全てのサポート言語において、表示崩れや翻訳漏れがないかを確認します。

## 2. テストの構成

### ユニットテスト (Jest)
- **対象:** `logic.js`, `db.js`, `utils.js` などの純粋なロジック。
- **実行:** `npm test`

### E2Eテスト (Playwright)
- **対象:** 複数タブ間の同期、設定変更、データのインポート/エクスポート、アニメーションの動作。
- **実行:** `npm run test:e2e`

### アニメーション評価
- **対象:** `src/js/animation/*.js`
- **内容:** 描画密度、変化率、および FG 回避ロジックの妥当性を機械的に評価します。
- **実行:** `npm run test:animation-eval`

## 3. 主要なテストケース

- **打刻ロジック:** タスクの開始、終了、一時停止が正しく IndexedDB に記録されること。
- **自動修復:** 異常終了後の再起動時に、整合性の取れないデータが修復されること。
- **同期:** あるタブで行った変更が、BroadcastChannel を通じて他のタブに即座に反映されること。
- **バックアップ:** File System Access API を使用したバックアップ・リストアが正常に機能すること。
