# 製品仕様書・設計指針: QuickLog-Solo (ミニマリスト向け・サイドパネル型作業メモツール)

## 1. プロジェクトのビジョン
- **コンセプト:** 「1秒で記録、1秒で集計、1秒で安心」。
- **持続可能性 (Sustainability) と長寿命な設計:** OSS のトレンドやライフサイクルに依存しない Vanilla JS による構築。外部依存を最小化し、10年後も今と同じように動作し続ける「持続可能性」を最上位の価値とします。
- **ターゲット:** 業務記録を負担に感じるが、ツールの透明性や安全性に厳しい技術者。
- **製品のポジショニング:** 本ツールは「作業メモツール」であり、エンタープライズ向けの「工数管理システム」とは一線を画す。高度な分析や外部連携を排し、個人の記録体験を最大化することに特化する。
- **設計思想・判断の背景:** 詳細については [AGENTS.md](../AGENTS.md) も併せて参照すること。

## 2. 設計思想と判断の背景 (Why)

本プロジェクトにおける技術的・設計的判断の理由は以下の通りです。

### 2.1. コア・フィロソフィー
- **持続可能性 (Sustainability):** 外部サーバーや API に依存せず、オフライン（ブラウザ内）で完結すること。ブラウザやタブが閉じられても、計測状態が確実に復元・継続されること。「レジリエンス（回復力）」という言葉よりも、システムの普遍性と自己完結性を強調する「持続可能性」を設計の最上位概念としています。
- **ミニマリズム (Minimalism):** 「必要十分（Less is More）」。過剰な機能追加を避け、特定のワークフロー（作業記録）の摩擦を最小化すること。
- **透明性 (Transparency):** データの保存先（IndexedDB）や通信が行われないことを明示し、ユーザー（技術者）が安心して利用できること。バックアップを実行した際には、ローカルファイルシステムにも保存されることを明確にしています。
- **長期保守性 (Longevity & Maintainability):** 外部フレームワークやライブラリを一切使用せず、標準的な Web API (Vanilla JS) のみで構築する「ピュアで一生モノの設計」。これにより、特定の OSS のライフサイクルやトレンドに左右されず、10年後もメンテナンス可能で動作し続けるコードを目指します。

### 2.2. 技術選定とアーキテクチャ
- **Vanilla JS & フレームワーク禁止 (OSS 非依存による長寿命化):** 依存関係の複雑化（Dependency Hell）と、ビルドツールのバージョンアップに伴うメンテナンスコストを排除するためです。OSS のライフサイクルやトレンド、破壊的変更に振り回されることなく、長期間安心して利用できる「持続可能性」を確保しています。また、ブラウザ拡張機能という特性上、セキュリティレビューの透過性を高め、ブラックボックスをなくす目的もあります。
- **IndexedDB & Local File System (Privacy First):** 外部への通信を CSP により技術的に強制遮断し、プライバシーを確保。データはブラウザ内の IndexedDB に保存され、バックアップ実行時にはローカルファイルシステムにも保存されます。
- **Vercel への配慮:** ルートの `index.html` が 404 になる「シャドウイング」を防ぐため、プロジェクトルートに `public/` ディレクトリを作成せず、アセットは `shared/assets/` に集約しています。
- **デプロイ・プロトコル (Vercel Build Bypass):** 拡張機能の ZIP パッケージをランディングページから直接配信するため、Vercel 内部のビルドシステムをバイパスし、GitHub Actions で生成された成果物をそのまま公開する設定（`vercel.json` の `outputDirectory` を `.` とし、`buildCommand` を空の操作にすることで実現）を採用しています。
- **DB・同期の分離 (Isolation):** URL パラメータ `?db=` を指定することで、独立した IndexedDB インスタンスを利用可能。状態同期（BroadcastChannel）にデータベース名を組み込むことで、複数インスタンス間での混線を防止します。

### 2.3. UI/UX 設計
- **Material 3 (M3) Tonal Palette:** 単一のアクセントカラーではなくトナルパレットを使用することで、アクセシビリティを確保しつつカテゴリごとの識別性を向上させています。
- **body への CSS 変数定義:** テーマ切り替え時に変数が確実に上書き・伝播されるようにするため、`:root` ではなく `body` に定義しています。
- **グリフの選定ポリシー (Privacy-first Glyph Selection):** 「Local Only」のポリシーを誤解させないため、クラウドを想起させる「雲（cloud）」や「同期（sync）」のアイコン使用を禁止し、物理ストレージをイメージさせる `hard_drive` などのアイコンを採用しています。
- **ページネーション (8x2 グリッド):** サイドパネルの限られたスペースで、ボタン配置を固定し、筋肉の記憶（マッスルメモリー）による素早い打刻を支援します。
- **タイポグラフィ:** 全ての言語で一貫したモダンな外観を維持し、OS 標準フォントへの不適切なフォールバックを防ぐため、Web フォントを徹底しています。
- **フォールスルー・パターン:** 背景アニメーションへの干渉と前面テキストの操作を両立させるため、コンテナに `pointer-events: none` を設定し、必要な要素のみ `auto` で有効化しています。

### 2.4. データ管理
- **40日間 & 100件表示:** パフォーマンス維持と「直近の振り返り」への特化のため。履歴表示を 100 件に絞ることで DOM 負荷を抑え、動作を軽快に保ちます。
- **終了ロジック (待機ログ):** 単純な「終了時刻の記録」ではなく、あえて「終了マーカー（待機ログ）」を記録するのは、日報作成時に「いつ作業が終わったか」を視覚的に明確にするためです。また、手動終了と自動終了（次のタスク開始による終了）を区別することで、計測の連続性を保ちつつ、ユーザーの明示的な意思表示（作業完了）を履歴に残せるようにしています。
- **重複マーカーの防止:** 停止ボタンの連打等により、同じタイムスタンプを持つ停止マーカー（`isManualStop: true` かつ `SYSTEM_CATEGORY_IDLE`）が重複して記録されるのを防ぐロジックを備えています。
- **内部文字列の永続性とローカライズ:** データ互換性のため、`__IDLE__` などの言語に依存しない ID を使用し、UI 表示時のみ翻訳関数を通しています。
- **セキュリティ標準 (textContent の徹底):** 動的な UI 翻訳やユーザー入力の表示において、`innerHTML` ではなく `textContent` を使用することで、XSS（クロスサイトスクリプティング）のリスクを最小化します。

### 2.5. 背景アニメーション (Visual Healing)
- **2分周期のサイクル:** 視覚的な変化を緩やかにし、集中を妨げないためです。
- **LCD ドットマトリクス・スタイル:** 最新のデザインの中にレトロな温かみと「1秒の重み」を表現しています。20種類以上の多彩なバリエーションを搭載しています。
- **パフォーマンス最適化標準 (TypedArray & Row-major):** 高負荷なシミュレーションでは、多次元配列を避け、1次元の `TypedArray` と行優先インデックス（`y * width + x`）を使用することでキャッシュ効率を高め、GC 負荷を低減します。
- **Web Worker によるサンドボックス化:** 外部モジュールの実行リスクからユーザーデータを保護し、UI の応答性を維持します。
- **負荷監視 (ガードレール):** 低スペック環境での安定性のため、200ms 以上の遅延が 20回連続した場合に自動停止します。また、起動直後の 3 秒間を「ウォームアップ期間」として監視対象外にしています。

## 3. システム構成 (What)
- **形態:** ブラウザ拡張機能 (Chrome/Edge: サイドパネル, Firefox: サイドバー)
- **技術:** HTML5, CSS3, JavaScript (Vanilla JS)。Web Worker によるアニメーションロジックのサンドボックス化。
- **ストレージ:** ブラウザ内 IndexedDB (Local Only)。
- **配布:** 各ブラウザストアまたはデベロッパーモードによるインストール。

## 4. 主要機能 (MVP)
### A. 打刻・記録ロジック
- カテゴリを選択して「開始」を押すと、現在時刻を打刻し、即座に履歴の先頭に表示される。
- 新しいタスクの「開始」により、前のタスクの終了時刻を自動記録。
- 終了ボタン（⏹）による終了時は、履歴の開始時刻を非表示にし、終了時刻のみを表示する特別なフォーマットを採用。
- 同時に実行できるタスクは常に1つのみ。
- **タブ間継続:** ブラウザのタブを切り替えてもサイドパネルの状態は維持され、計測が継続される。
- **状態表示:** 実行中: ▶ (Green), 待機中/一時停止中: ⏸ (Orange - 点滅あり), 終了済み: ⏹ (Red)。
- **アラーム・通知機能:** 指定した時刻にメッセージを表示し、作業を自動的に「終了」「一時停止」「開始」できます。
    - **確認待ちモード:** 各アラームに「確認」オプションを設定できます。これが有効な場合、通知に「了解」「閉じる」ボタンが表示され、ユーザーが「了解」を押すまでアクションの実行を保留し、通知を維持します。また、「通知のみ」のアラームで確認モードが有効な場合は「閉じる」ボタンのみが表示されます。
    - **デフォルトアラーム:** 初期状態で 23:59 に終了するアラームが設定されており、止め忘れを防止して翌日の記録をクリーンに開始できるようサポートします。

### B. カテゴリ管理
- ユーザーがカテゴリを追加・削除・編集可能。設定は永続化される。
- ドラッグ＆ドロップによる表示順序の変更が可能。
- ページネーション（1ページ16項目、8x2グリッド）をサポート。マウスホイールで切り替え可能。
- カテゴリデータの NDJSON 形式でのインポート/エクスポートが可能。
- **業務カテゴリ・エディタ (v0.32.0):** 広い画面での編集を可能にするウェブベースのエディタ（Vercel上でホスト）。拡張機能本体を更新することなく改善を迅速に届けるため、ウェブ版への移行を行いました。データ連携はクリップボード（NDJSON）を介して行います。本体アプリから遷移した際は、不要な戻るリンクを非表示にする条件付きナビゲーションを維持。
- **一括編集時のタグ管理:** 複数カテゴリが選択されている場合、エディタはそれらすべてに共通するタグ（積集合）のみを表示し、アルファベット順にソートします。一括追加時は既存の順序を維持しつつ重複を避けて末尾に追加、一括削除時は選択されたすべてのカテゴリから対象タグを削除します。

### C. 出力機能 (クリップボードコピー)
- ヘッダーのボタン（📋, 📊）から、日報形式やタグ別の工数集計結果のコピーが可能。
- **日報コピー:** Markdown, Wiki, HTML Table, CSV, Text形式をサポート。時刻アジャスト（5/10分単位の丸め）が可能。
- **経過時間フォーマット:** `formatLogDuration` により、1時間以上の場合は「1h 5m」（10分未満はスペースあり）、「1h15m」（10分以上はスペースなし）という視覚的なリズムを持った形式で出力されます。
- **タグ集計:** 選択した日付のログから、カテゴリに紐付いたタグごとの合計時間を算出。
- **コピー操作の標準化:** クリック時にトースト通知のみを表示し、ボタン自体のテキストは変更しません。ホバー時は M3 トナル・インバージョンを適用。

### D. データライフサイクルとバックアップ
- **保持期間:** 直近40日間。起動時に自動削除。
- **ローカルファイルバックアップ:** File System Access API を使用した、特定ディレクトリへの履歴の手動バックアップ。バックアップ実行時に、ログは1日1ファイル、カテゴリは `categories.ndjson`（NDJSON形式による行ベースの手動並べ替えを考慮）、設定は `settings.json`（可読性重視のJSON形式）として保存されます。
- **手動保守:** ログデータのCSVエクスポート/インポート（設定の一般タブ）、カテゴリデータのNDJSONエクスポート/インポート（設定のカテゴリタブ）が可能です。
- **3段階のデータインポートバリデーション:**
    - **レベル 1 (Fatal):** ファイルが空、または全体が破損しており読み込み不可能な場合。エラーダイアログを表示し中断します。
    - **レベル 2 (Partial):** 一部の行が JSON として正しくない場合。ユーザーに「正常な行のみ読み込むか、中断するか」を選択させます。
    - **レベル 3 (Field):** JSON 自体は正しいが、カテゴリ名が長すぎる、または色が不正な場合。ユーザーに「デフォルト値を適用するか、スキップするか、中断するか」を選択させます。

### E. 背景アニメーション
- **レイヤー構造 (FG/BG):** 前面 (FG) と背景 (BG) を定義し、視認性と表現力を両立。
- **透過性の維持:** アニメーションがアクティブな際、前面要素の背景色を CSS `!important` で強制的に透過させます。

## 5. UI/UX デザイン・詳細仕様
- **レスポンシブ:** サイドパネルに最適化された垂直レイアウト。
- **Material 3:** Google の Material 3 に完全準拠。IconButtons は Material Symbols を使用。
- **設定 UI (v0.31.0):** 設定画面のタブを Material Symbols によるアイコン表示に刷新。`data-i18n-title` による多言語対応ツールチップを完備し、省スペース化と視認性を両立。
- **堅牢なレイアウト:** 多言語対応のため、重要な UI 要素には `text-overflow: ellipsis` を適用し、レイアウト崩れを防止。
- **履歴の再現性:** ログに打刻時点のカテゴリ色を保存し、カテゴリ削除後も当時の色で表示。
- **デモ履歴生成:** ログが空の状態で起動した際、擬似的な業務履歴を自動生成。
- **透明性の明示:** 「About」タブにて、保存先や通信仕様 (Local Only) を明示し、IndexedDB 内の統計情報を表示。

## 6. セキュリティと運用
- **セキュリティポリシー:** `textContent` による XSS 対策、入力文字列の最大長制限 (50文字)、予約語の保護。
- **運用上の制約:** 外部 API 不使用。シンプルさと軽量動作を最優先。
- **パッケージング:** `scripts/create_package.py` による自動化。
- **二系統ビルド戦略:** 配布用の「Release版（青）」と検証用の「Dev版（橙）」を同時に生成。
- **ルートディレクトリのクリーンネス:** CI において、AI エージェント等による一時的なスクリプトの混入を厳格にチェック（`check_root_files.py`）。

- **構成:** 1ページ構成。最新版ダウンロードリンクとブラウザでのプレビュー機能を備える。

## 8. 品質管理と開発プロセス
- **テスト:** Jest (単体), Playwright (E2E), アニメーション評価テスト。`i18n_coverage.test.js` により全言語の翻訳整合性を保証。
- **データ整合性:** `initDB` 実行時に、終了時刻のないタスクや古い待機ログを自動修復します。
- **日本語用語の標準化:** ユーザーの混乱を防ぐため、「カテゴリ」（短母音）、「インポート/エクスポート」、「終了」（Stopではなく完了の意味）、「時刻アジャスト」、「日付変更時の”作業終了"自動記録」といった用語を標準化しています。
- **ディレクトリ構成:** 詳細は `README_DEV.md` を参照。メインアプリは `projects/app/`, サブプロジェクトは `projects/`, 共通コードは `shared/`, スクリプトは `scripts/` に集約されています。
- **ドキュメント:** PR におけるやり取りはすべて**日本語**。セッション内の判断は随時 spec.md に反映する。
- **バージョニング:** Conventional Commits に基づく自動採番 (`scripts/bump_version.py`)。
- **ルートディレクトリのクリーンネス:** CI において、AI エージェント等による一時的なスクリプトの混入を厳格にチェック（`check_root_files.py`）。
- **パス・ポータビリティ標準:** ローカル、Vercel、拡張機能の全環境で動作させるため、アセット参照は常に相対パス（先頭スラッシュなし）を使用します。

## 9. QL-Animation Studio (β版)
ブラウザ上でアニメーションロジックを作成・テスト・ダウンロード可能な開発環境。
- **特徴:** 行番号・ハイライト付きエディタ、スピード調整付きプレビュー、除外領域シミュレータ。
- **レイアウトの安定性:** アニメーション選択時に UI がガタつくのを防ぐため、説明文エリアの高さは 3.5 行分に固定（`calc(1.4em * 3.5)`）されています。
- **単一の開発パス (Single Development Path):** 「既存のサンプルをベースに開発する」という単一のパスに集中させることで、開発者の迷いを減らします。スニペット機能ではなく `devOnly: true` の実例モジュールを通じて学習を促進します。
- **FG回避 (exclusionStrategy):** `'mask'`, `'jump'`, `'freedom'` の 3 つから選択可能。
- **開発用モジュール (devOnly):** `devOnly: true` のモジュールはリリースパッケージから除外されるが、Studio や紹介ページでは利用可能。
- **Studio UI (v0.31.0):** ツールチップの多言語対応 (`data-i18n-title`) を実装。再生中のスクラブ操作（巻き戻し/先送り）において、`isScrubbing` フラグと `engine.isDrawPending` による並行制御を行い、UI の応答性と描画の正確性を両立。
