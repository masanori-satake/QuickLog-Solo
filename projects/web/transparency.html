透明性と安全性の維持

QuickLog-Soloは、技術的な裏付けに基づいた透明性と安全性を最も重視しています。GitHubや専門知識に不慣れな方でも、私たちの取り組みを安心してご理解いただけるよう、ここで詳しく説明します。

code

1. 外部ライブラリへの依存ゼロ

QuickLog-Soloは、外部ライブラリ（OSS）を一切使用せず、「純粋なJavaScript」のみで構築されています。これは「ブラックボックス」を排除することを意味します。すべてのコードが公開されており、第三者による検証が容易です。

check_circle 外部依存関係なし
security

2. 継続的な自動セキュリティ監査

GitHub Actionsという仕組みを利用し、開発の全工程でコードの整合性と安全性を自動的にチェックしています。AIなどによる意図しないコードの混入や、脆弱性のリスクを常に監視し、高い基準を維持しています。

published_with_changes 自動監査パス
search_check

3. Google OSV-Scanner による監査

Googleが提供する脆弱性スキャンツール「OSV-Scanner」を導入しています。これにより、万が一開発環境などのツールに既知の脆弱性が発見された場合でも、即座に検知して対応できる体制を整えています。

verified Google OSV 監査済み
lock

4. ローカル完結のプライバシー

あなたの記録はすべてブラウザ内にのみ保存されます。外部サーバーへの送信は一切行われず、私たち開発者であってもあなたのデータを見ることはできません。この事実は、ブラウザの開発者ツールで誰でも確認可能です。
